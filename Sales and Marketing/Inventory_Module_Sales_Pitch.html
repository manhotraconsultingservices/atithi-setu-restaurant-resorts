4Three features owners say change everything

Auto-deduction is invisible

Cashier doesn't know inventory is happening. Place 50 butter chickens → 10 kg chicken, 1.5 kg butter, 2.5 L cream silently subtract from stock. Cancellations auto-revert. Concurrent orders use atomic SQL — never race.

9 AM auto-PO is owner's time machine

Wakes up to 3 ready-to-send draft POs, grouped by supplier, with quantities derived from par level − stock − on-order. One tap → supplier gets a PDF on email. 30 min/day reclaimed across the year = 180 hours back.

Variance report = lie detector

After every weekly count, see exactly which ingredients had the biggest negative variance, monetised. "₹3,400 paneer unaccounted for" — converts hunch into number, the conversation with the chef changes overnight.

6What owners ask before signing up

"My menu has 200 items — too much to set up."
Built-in CSV import takes 199 items in 5 minutes. Recipe template downloads pre-populated with your menu. Most owners map their top-30 dishes and let the rest go un-tracked initially.
"What if my chef changes the recipe?"
Edit the recipe — old version is preserved with effective_to timestamp. Historical orders keep their old recipe forever. Going-forward orders use the new one. No data loss.
"Will it slow down order taking?"
Auto-deduction is fire-and-forget — runs after the order is saved. Even if deduction fails, the order is unaffected. Zero added latency at point-of-sale.
"What if I don't have a default supplier?"
Smart-PO preview groups items by default supplier — items without one are flagged and skipped. Set a default, click Smart PO again, draft POs appear. No data lost.
